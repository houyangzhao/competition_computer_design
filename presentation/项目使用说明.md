## 二、Web 功能操作

### 2.1 浏览古建筑档案

点击导航栏「探索」进入档案列表。

- **已完成重建**的建筑：点击卡片进入三维查看页
- **待重建**的建筑：显示"数字基因缺失"，可点击"贡献照片"参与众包

**三维模型操作（进入建筑详情页后）：**

| 操作 | 效果 |
|------|------|
| 鼠标左键拖拽 | 旋转视角 |
| 鼠标滚轮 | 缩放 |
| 鼠标右键拖拽 | 平移 |

右侧面板可切换「建筑构件知识」和「AI 讲解」（AI 讲解功能待上线）。

---

### 2.2 上传照片自动重建

1. 点击导航栏「重建」
2. 填写建筑名称（如：岳麓书院大门）
3. 点击上传区域或拖拽照片（支持 JPG/PNG，建议 20–300 张）
4. 点击「开始重建」，页面自动显示进度

**重建阶段说明：**

| 阶段 | 在做什么 | 大概时间 |
|------|----------|----------|
| 筛选图片 | 过滤重复/质量差的图片 | < 1 分钟 |
| COLMAP 重建 | 分析每张照片的拍摄位置 | 5–20 分钟 |
| 3DGS 训练 | 用神经网络生成三维模型 | 10–40 分钟 |
| 完成 | 可点击「查看模型」 | — |

5. 重建完成后点击「查看模型」即可在浏览器里查看三维结果

## 三、如何拍出适合重建的照片

这是决定重建效果的最关键因素。

**✅ 好的拍摄方式：**
- 围绕建筑走一圈，每走 1–2 步拍一张
- 覆盖多个高度（站立拍、抬头拍、弯腰拍）
- 相邻照片重叠 60% 以上
- 光线均匀，避免正午强烈阴影
- 同一建筑 50–200 张为宜

**❌ 常见问题：**
- 只站在一个位置拍多张 → COLMAP 无法估计空间关系
- 运动模糊 → 特征点提取失败
- 大面积纯色墙面（无纹理） → 匹配失败
- 室内光线不足 → 噪点多，训练效果差


### 4.1 用 Gemini / Claude 阅读源码的方法

把以下文件喂给 AI，然后提问：

**理解整体流程（先看这个）：**
```
vendor/gaussian-splatting/train.py          ← 训练主循环（约400行，最重要）
vendor/gaussian-splatting/scene/__init__.py ← 场景数据加载
```

**理解高斯表示（核心数据结构）：**
```
vendor/gaussian-splatting/scene/gaussian_model.py   ← 高斯模型定义
```

**理解渲染过程：**
```
vendor/gaussian-splatting/gaussian_renderer/__init__.py
```

**理解 COLMAP 在干什么（用 Python 调用的接口）：**
```
vendor/gaussian-splatting/convert.py        ← 调用 COLMAP 的脚本
reconstruction/reconstruct.sh              ← 我们自己的重建流程
```

### 4.3 关键概念速查

| 术语 | 中文 | 一句话解释 |
|------|------|-----------|
| SfM (Structure from Motion) | 运动恢复结构 | 从多张照片自动计算每张照片的拍摄位置 |
| Camera Pose | 相机位姿 | 拍照时相机在三维空间中的坐标和朝向 |
| Sparse Point Cloud | 稀疏点云 | COLMAP 输出的初始三维点，是 3DGS 的"种子" |
| 3D Gaussian | 三维高斯椭球 | 场景中每个"粒子"，有位置、大小、颜色、透明度 |
| Spherical Harmonics (SH) | 球谐函数 | 描述颜色随视角变化的数学工具 |
| Densification | 致密化 | 训练中自动在细节区域增加粒子数量 |
| Opacity Culling | 透明度剪枝 | 训练中删除接近透明的无效粒子 |
| .splat | splat 文件格式 | 存储所有高斯椭球参数的二进制格式，供网页渲染 |
| Radiance Field | 辐射场 | 用数学函数描述三维空间中光的分布 |
