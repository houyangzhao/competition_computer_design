# 筑忆 — 技术实现梳理（给写PPT/文书的队友）

---

## 一、项目是什么

**"筑忆"** 是一个全民参与的濒危古建筑数字抢救平台。

**核心用法**：用户用手机拍多角度照片 → 上传到平台 → 平台自动生成古建筑的 3D 模型 → 任何人可以在浏览器里实时漫游这个 3D 模型。

### 3.2 3D 渲染（已验证可用）

浏览器直接加载 `.splat` 文件，无需插件，基于 WebGL 实时渲染 3D Gaussian Splatting 模型，流畅交互漫游。

### 3.3 真实 3D 重建管线（已在真实 GPU 服务器上跑通）

这是技术亮点，完整走通了以下流程：

```
手机照片（JPG）
    ↓
COLMAP SfM（GPU加速）
  ① 特征提取（GPU SIFT）
  ② 跨图像特征匹配
  ③ Bundle Adjustment → 自动估计每张照片的相机位置和朝向
  ④ 输出稀疏点云 + 相机位姿
    ↓
3D Gaussian Splatting 训练
  输入：稀疏点云 + 相机位姿
  输出：point_cloud.ply（约 213 MB）
  训练时间：30,000 步约 4 小时（RTX 4080 SUPER）
    ↓
格式转换（自写 Python 脚本）
  PLY → .splat（约 27 MB）
  每个 Gaussian 32 字节：位置 + 尺度 + 颜色 + 旋转
    ↓
Web 端加载渲染
```

**实际成果（故宫午门场景）**：

| 指标 | 数值 |
|---|---|
| 输入照片 | 29 张（9504×6336） |
| 全部注册成功 | 29/29 张 |
| Gaussian 数量 | 897,628 个 |
| PSNR | 25.6 dB |
| .splat 文件大小 | **27.4 MB** |
| 浏览器渲染 | 流畅交互 |

## 四、关键技术原理（便于PPT讲解）

### 为什么用户不需要专业设备？

COLMAP 的 SfM（运动恢复结构）算法会从普通照片中**自动计算**每张照片的拍摄位置和角度，用户只需拍摄多角度、有 70% 以上重叠的照片即可，无需 GPS 坐标或任何专业测绘设备。

### 3D Gaussian Splatting 是什么？

2023 年 SIGGRAPH 最佳论文技术。用数十万个半透明"椭球体"（Gaussian）来表示三维场景，每个椭球记录位置、形状、颜色和透明度。相比传统 NeRF，渲染速度快 10 倍以上，可实现**实时**浏览器交互。

### 天地图为什么特别提？

国内竞赛合规要求地图必须有审图号（中国地图），天地图是国家测绘局官方地图服务，自带合规资质，避免使用高德/Google地图的合规风险。

## 五、两种使用模式（产品亮点）

| 模式 | 流程 |
|---|---|
| **个人重建** | 用户注册 → 上传照片 → 等待自动重建 → 保存到"我的模型" |
| **公众众包** | 平台发起公共项目（如"故宫全景"）→ 人人贡献照片 → 汇入公共资源池 → 合并重建 → 公共 3D 模型展示 |

众包模式的价值：一个人可能只能拍到故宫某个角度，但数百人贡献的照片可以拼出完整全景——这是平台最核心的"数据涌现"价值。
