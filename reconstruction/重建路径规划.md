# 故宫3DGS重建路径规划

## 目标

在 AutoDL GPU 实例上，基于 CULTURE3D 数据集的故宫（紫禁城）子场景，训练出一个可在
Web 端实时渲染的 `.splat` 文件，作为"筑忆"平台首页的展示模型。

---

## 代码库选型（核心决策）

### 管线总览

```
原始照片
  │
  ▼
COLMAP 3.9+          ← 相机位姿估计（SfM）+ 稀疏点云
  │  如果 CULTURE3D 已提供 COLMAP 结果，跳过此步
  ▼
3DGS 训练（二选一）
  ├── 路线A：graphdeco-inria/gaussian-splatting   ← 图像<500张 / 局部小场景
  └── 路线B：graphdeco-inria/hierarchical-3d-gaussians  ← 图像>500张 / 全故宫大场景
  │
  ▼
输出 point_cloud.ply
  │
  ▼
convert_ply_to_splat.py  ← 自写转换脚本
  │
  ▼
taihedian.splat          ← 放入 frontend/public/models/
```

---

### 库一：COLMAP

- **仓库**：https://github.com/colmap/colmap（版本 3.9+）
- **作用**：从无序照片自动估计相机内外参（SfM），输出稀疏点云，作为 3DGS 的初始化输入
- **使用方式**：3DGS 官方仓库内置了 `convert.py` 脚本，调用 COLMAP 命令行，一行搞定
- **如果 CULTURE3D 已提供 COLMAP 结果**（`sparse/0/` 目录），完全跳过，直接进 3DGS

---

### 库二A：graphdeco-inria/gaussian-splatting（原版）

- **仓库**：https://github.com/graphdeco-inria/gaussian-splatting
- **适用场景**：图像子集 **< 500 张**，只重建太和殿局部
- **优势**：
  - 最权威的原始实现，文档齐全，坑少
  - 输出标准 PLY 格式，与所有下游工具兼容
  - CULTURE3D 论文本身就是用此方法评测的，复现有据可查
- **劣势**：
  - 显存占用较高，图像多时可能 OOM
  - 不支持超大场景分块
- **训练命令**：
  ```bash
  python train.py -s /root/data/taihedian -m /root/output/result \
    --iterations 7000       # demo质量，~45分钟
  ```
- **RTX 3090 显存预估**：150张图 ≈ 10-14GB，300张图 ≈ 16-20GB

---

### 库二B：graphdeco-inria/hierarchical-3d-gaussians（分层版，备选）

- **仓库**：https://github.com/graphdeco-inria/hierarchical-3d-gaussians
- **论文**：SIGGRAPH 2024，专为超大户外场景设计
- **适用场景**：CULTURE3D 故宫数据图像量 **> 500 张** 且无法有效裁剪子集时
- **优势**：专门解决大场景显存/训练时间问题，支持分块重建后合并
- **劣势**：
  - 配置更复杂，依赖更多（需要额外 `hierarchy-rasterizer`）
  - 输出是分层结构，转 .splat 需要额外处理
  - 调试成本更高
- **策略**：**优先尝试路线A，OOM 或质量极差时再切换**

---

### 库三：格式转换

- **输入**：`point_cloud.ply`（3DGS 训练输出）
- **输出**：`.splat`（Web 端 @mkkellogg/gaussian-splats-3d 可直接加载）
- **实现方式**：自写约50行 Python 脚本（见下文 `convert_ply_to_splat.py`）
- **格式说明**：`.splat` 是二进制格式，每个 Gaussian 占 32 字节：
  ```
  xyz(3×float32) + scale(3×float32) + color(4×uint8) + rot(4×uint8)
  ```
- **注意**：`@mkkellogg/gaussian-splats-3d` 也支持直接加载 `.ply`，若转换出问题可先试 PLY

---

## 决策树（下载数据后执行）

```
下载 CULTURE3D 故宫数据
  │
  ├─ 是否已有 sparse/ COLMAP 结果？
  │    ├─ 是 → 跳过 COLMAP，直接 Step 3
  │    └─ 否 → 执行 Step 2（COLMAP）
  │
  ├─ 故宫图像总量？
  │    ├─ 可筛出 <300 张太和殿子集 → 用路线A（原版 gaussian-splatting）
  │    └─ 无法有效筛选 / 全部图像量极大 → 用路线B（hierarchical）
  │
  └─ 训练时 OOM？
       ├─ 是 → 减少图像数 或 切换路线B
       └─ 否 → 继续，7000步先看效果
```

## Step by Step 操作

### Step 0：开机确认环境

```bash
# AutoDL 推荐镜像：PyTorch 2.1 / CUDA 11.8 / Ubuntu 22.04
# GPU：RTX 3090 或 4090
nvidia-smi                     # 确认 GPU 可用
python -c "import torch; print(torch.cuda.is_available())"
```

### Step 1：安装依赖

```bash
# COLMAP
apt-get update && apt-get install -y colmap

# 路线A：原版 3DGS
cd /root
git clone --recursive https://github.com/graphdeco-inria/gaussian-splatting.git
cd gaussian-splatting
pip install plyfile tqdm
pip install submodules/diff-gaussian-rasterization
pip install submodules/simple-knn

# 路线B（备用，路线A失败时再装）
# git clone --recursive https://github.com/graphdeco-inria/hierarchical-3d-gaussians.git
```

### Step 2：下载 CULTURE3D 数据

```bash
pip install gdown
# 从 CULTURE3D GitHub README 找到故宫场景的 Google Drive 链接
gdown --folder <GoogleDrive链接> -O /root/data/forbidden_city_raw/

# 查看数据结构
ls /root/data/forbidden_city_raw/
# 预期：images/ 或 input/ 目录，可能包含 sparse/ 目录
```

### Step 3：准备输入目录

```bash
mkdir -p /root/data/taihedian/input

# 情况A：CULTURE3D 未提供 COLMAP 结果
# → 筛选太和殿图像子集（约150-300张）放入 input/
# → 筛选依据：文件名、GPS EXIF、或手动浏览挑选

# 情况B：CULTURE3D 已提供 sparse/
# → 直接复制整个场景目录，跳过 COLMAP
```

### Step 4：COLMAP（情况A才执行）

```bash
cd /root/gaussian-splatting
python convert.py -s /root/data/taihedian
# 完成后应有 /root/data/taihedian/sparse/0/
```

### Step 5：3DGS 训练（路线A）

```bash
cd /root/gaussian-splatting
python train.py \
  -s /root/data/taihedian \
  -m /root/output/taihedian_7k \
  --iterations 7000

# 若显存充足且时间允许，提升至30000步：
# python train.py -s /root/data/taihedian -m /root/output/taihedian_30k --iterations 30000
```

### Step 6：PLY 转 .splat

```bash
python /root/convert_ply_to_splat.py \
  /root/output/taihedian_7k/point_cloud/iteration_7000/point_cloud.ply \
  /root/output/taihedian.splat

# 验证文件大小（正常应在 50-300MB 之间）
ls -lh /root/output/taihedian.splat
```

### Step 7：下载到本地

通过 AutoDL 的文件管理面板或 `scp` 下载 `taihedian.splat`，
放入 `frontend/public/models/`，并更新 `buildings.ts`：

```ts
modelPath: '/models/taihedian.splat'
```

## 风险与备选

| 风险 | 应对 |
|------|------|
| CULTURE3D 数据无法下载 | 用 Wiki Commons / 故宫官网多角度照片自建数据集 |
| 太和殿重建效果差（俯视角为主） | 改重建 CULTURE3D 中其他场景（金字塔 / 剑桥） |
| 路线A 显存 OOM | 减少图像数至100张以下；或切路线B |
| .splat 转换失败 | viewer 直接加载 .ply（@mkkellogg 支持） |
