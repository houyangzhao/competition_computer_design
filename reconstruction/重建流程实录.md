# 3DGS 重建流程实录

基于 CULTURE3D 数据集，在 AutoDL 服务器上完成从原始照片到 `.splat` 文件的完整流程。

## 环境信息

- GPU：NVIDIA RTX 4080 SUPER（32GB VRAM）
- CUDA：12.1
- OS：Ubuntu 22.04
- Python：3.12（conda base）
- PyTorch：2.3.0+cu121

---

## Step 0：数据下载

使用 rclone 从 Google Drive 下载场景数据（需提前配置好 `gdrive:` remote）。

```bash
# 列出目标文件夹内容（先确认结构）
rclone lsd "gdrive:" --drive-root-folder-id <FOLDER_ID> --max-depth 2

# 下载到 autodl-tmp
mkdir -p /root/autodl-tmp/<scene_name>
rclone copy "gdrive:" /root/autodl-tmp/<scene_name> \
  --drive-root-folder-id <FOLDER_ID> \
  --transfers 4 --retries 5
```

**注意**：CULTURE3D 数据集只提供 `Input Image/`（原始照片）和 `RealityCapture/`（RC 格式 SfM 数据），**没有 COLMAP 格式数据**，需要自己跑 COLMAP。

准备场景目录：

```bash
mkdir -p /root/autodl-tmp/<scene_name>/input
cp "/root/autodl-tmp/<scene_name>/Input Image/"*.JPG /root/autodl-tmp/<scene_name>/input/
```

## Step 2：从源码编译 CUDA 版 COLMAP

**原因**：apt 安装的 COLMAP 3.7 编译时无 CUDA，只能跑 CPU SIFT；官方 release 只提供 Windows 二进制，Linux 必须自行编译。

### 2.1 安装编译依赖

```bash
apt-get install -y \
    git cmake ninja-build build-essential \
    libboost-program-options-dev libboost-filesystem-dev \
    libboost-graph-dev libboost-system-dev \
    libeigen3-dev libflann-dev libfreeimage-dev libmetis-dev \
    libgoogle-glog-dev libgflags-dev libsqlite3-dev \
    libceres-dev libcgal-dev libglew-dev \
    libgl-dev libegl-dev libglx-dev libgles-dev libglvnd-dev \
    xvfb
```

### 2.2 克隆并修改 CMake

```bash
cd /root/Code
git clone https://github.com/colmap/colmap.git
cd colmap
git checkout 3.11.1
```

**必须修改** `cmake/FindDependencies.cmake`，去掉 GUI OFF 强制关闭 OpenGL 的逻辑（第 195-205 行）：

```cmake
# 原代码（有问题）：
if(OPENGL_ENABLED)
    if(NOT GUI_ENABLED)
        message(STATUS "Disabling GUI also disables OpenGL")
        set(OPENGL_ENABLED OFF)   # ← 这行导致 SiftGPU 无法用 GPU
    ...

# 改为：
if(OPENGL_ENABLED)
    add_definitions("-DCOLMAP_OPENGL_ENABLED")
    message(STATUS "Enabling OpenGL support (headless)")
else()
    message(STATUS "Disabling OpenGL support")
endif()
```

### 2.3 编译安装

RTX 4080 SUPER 的 CUDA arch 是 89（Ada Lovelace）：

```bash
mkdir -p /root/Code/colmap/build && cd /root/Code/colmap/build

cmake .. -GNinja \
    -DCMAKE_CUDA_ARCHITECTURES="89" \
    -DCMAKE_BUILD_TYPE=Release \
    -DCMAKE_INSTALL_PREFIX=/usr/local \
    -DGUI_ENABLED=OFF \
    -DOPENGL_ENABLED=ON

ninja -j$(nproc)
ninja install
```

验证：

```bash
colmap -h | head -2
# 应输出：COLMAP 3.11.1 ... (Commit ... with CUDA)
```

## Step 3：COLMAP SfM（GPU）

使用 gaussian-splatting 提供的 `convert.py`，配合 `xvfb-run` 提供虚拟显示（SiftGPU 需要 OpenGL context，即使是 headless GPU 也要通过 Xvfb 提供）：

```bash
cd /root/Code/gaussian-splatting
xvfb-run -a python convert.py -s /root/autodl-tmp/<scene_name>
```

`convert.py` 会自动执行：
1. `feature_extractor`（GPU SIFT）
2. `exhaustive_matcher`（GPU matching）
3. `mapper`（bundle adjustment，CPU）
4. `image_undistorter`

### ⚠️ 多模型问题

mapper 有时会输出多个稀疏模型（`sparse/0`, `sparse/1`, ...），`convert.py` 默认用 `sparse/0`，但最大的模型可能在其他编号。

**必须检查**：

```bash
for d in /root/autodl-tmp/<scene_name>/distorted/sparse/*/; do
    echo "$d:"
    colmap model_analyzer --path "$d" 2>&1 | grep "Registered images"
done
```

如果主模型不在 `sparse/0`，手动重跑 undistortion：

```bash
# 清除错误的 undistortion 结果
rm -rf /root/autodl-tmp/<scene_name>/sparse \
       /root/autodl-tmp/<scene_name>/images \
       /root/autodl-tmp/<scene_name>/stereo

# 用正确的模型（如 sparse/1）
colmap image_undistorter \
    --image_path /root/autodl-tmp/<scene_name>/input \
    --input_path /root/autodl-tmp/<scene_name>/distorted/sparse/1 \
    --output_path /root/autodl-tmp/<scene_name> \
    --output_type COLMAP
```

undistortion 默认输出的 `sparse/` 下文件没有 `0/` 子目录，需要整理：

```bash
mkdir -p /root/autodl-tmp/<scene_name>/sparse/0
mv /root/autodl-tmp/<scene_name>/sparse/*.bin \
   /root/autodl-tmp/<scene_name>/sparse/0/
```

完成后目录结构应为：

```
<scene_name>/
├── images/        ← undistorted 图像
├── sparse/0/      ← cameras.bin + images.bin + points3D.bin
└── input/         ← 原始图像（保留备用）
```

## Step 4：3DGS 训练

```bash
cd /root/Code/gaussian-splatting
python train.py \
    -s /root/autodl-tmp/<scene_name> \
    -m /root/autodl-tmp/<scene_name>/output_<N>k \
    --iterations <N>000 \
    --save_iterations <N>000 \
    --test_iterations <N>000 \
    -r 2 \
    --densify_until_iter $(( N * 1000 * 2 / 3 )) \
    --densify_grad_threshold 0.0001
```

**分辨率参数 `-r` 说明**（32GB VRAM 参考）：

| `-r` 值 | 图像尺寸 | 速度 | 质量 |
|---------|---------|------|------|
| 1 | 原始全分辨率 | 很慢 | 最高 |
| 2 | 1/2（推荐） | 中等 | 高 |
| 4 | 1/4 | 快 | 中 |
| 默认 | 短边≤1.6K | 快 | 低 |

**典型耗时（RTX 4080 SUPER，`-r 2`）**：

| 迭代数 | 时间 | 说明 |
|--------|------|------|
| 7,000  | ~6 分钟 | 偏少，质量不足 |
| **10,000** | **~10 分钟** | **标准选择** |
| 30,000 | ~4 小时 | 过长，收益边际递减 |

## Step 5：PLY → .splat 转换

```bash
python3 /root/Code/competition_computer_design/reconstruction/convert_ply_to_splat.py \
    /root/autodl-tmp/<scene_name>/output_<N>k/point_cloud/iteration_<N>000/point_cloud.ply \
    /root/autodl-tmp/<scene_name>/<scene_name>.splat \
    --transform outdoor_arch
```

**必须加 `--transform outdoor_arch`**，否则使用默认 `x180`，坐标系与 `compute_camera_settings.py` 不一致，导致相机起点/朝向全部错乱。
